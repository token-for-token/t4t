Token4Token

Decentralized AI inference marketplace on Gnosis Chain and Ethereum Swarm.

How it works

  1. Providers run a container

    Anyone with a GPU and an OpenAI-compatible backend (Ollama, vLLM, llama.cpp…) runs the T4T provider container. It registers on-chain, stakes xBZZ, and publishes the models it serves with per-million-token prices.

  2. Clients post jobs

    Clients run the T4T client container, which exposes a normal OpenAI /v1/chat/completions endpoint locally. It picks a provider from the on-chain directory, escrows the maximum payment, and sends the encrypted request over Swarm PSS.

  3. Provider serves, gets paid on delivery

    The provider runs inference locally, ships the response back over Swarm, and claims payment from escrow. If the deadline lapses, the client gets a refund. Repeated failures slash the provider’s stake.

  4. No middleman, no API key

    No central API gateway, no marketplace operator, no rate limit. Selection, payment, reputation, and liveness all live on-chain.

Full protocol: docs/spec.md.
